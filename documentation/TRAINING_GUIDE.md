# Training Guide

## Goal

This project now has two different training paths:

1. train the `YOLO` detector
2. train the `SVSP` direction model

They solve different problems.

## Training Path 1: YOLO Detector

YOLO is used for:

- detecting person or object
- producing bounding boxes

### Recommended YOLO workflow

1. collect images from your real robot camera
2. auto-label images
3. manually correct labels
4. fine-tune YOLO
5. set `vision.model_path` to the trained weights

### Collect raw images

```powershell
python scripts/collect_data.py --out data/raw_images
```

### Auto-label images

```powershell
python scripts/auto_label.py --images data/raw_images --out data/auto_dataset --classes person
```

### Train YOLO

```powershell
python main.py train data/auto_dataset/data.yaml --base-model yolov8n.pt --epochs 100 --name robot_target_model
```

### Use the trained YOLO weights

Update config:

```yaml
vision:
  model_path: models/trained/robot_target_model/weights/best.pt
```

## Training Path 2: SVSP Direction Model

SVSP is used for:

- reading bbox history
- predicting:
  - `left`
  - `right`
  - `forward`
  - `backward`
  - `stationary`

### Current SVSP implementation

The first version of SVSP is trained from synthetic bounding-box trajectories generated by code.

This allows:

- quick bootstrapping
- full code-driven training
- saving a working `svsp.pt`

It is a practical first step, not the final best production model.

### Train SVSP

```powershell
python main.py train-svsp --output models/svsp.pt
```

### Advanced SVSP training options

```powershell
python main.py train-svsp --output models/svsp.pt --samples 3000 --sequence-length 8 --epochs 150
```

## Recommended Real-World Evolution For SVSP

The current synthetic training is useful, but the long-term best path is:

1. collect real target trajectories from camera sessions
2. store bbox sequences over time
3. label motion direction
4. train SVSP on real sequences

That will make the motion model stronger in real environments.

## Detector Training vs Motion Training

### YOLO

Trains on:

- images
- bounding box labels

Output:

- detector weights

### SVSP

Trains on:

- sequences of bbox features
- direction labels

Output:

- motion model weights

## Suggested Training Order

The best order is:

1. get YOLO detection working well
2. get target selection and tracking stable
3. train SVSP for direction prediction
4. later add gesture-specific AI if needed

## API-Based SVSP Training

Start API:

```powershell
python main.py api --mock
```

Train SVSP:

```powershell
curl -X POST http://localhost:8000/api/v1/train/svsp
```

Load an existing model:

```powershell
curl -X POST http://localhost:8000/api/v1/model/svsp/load -H "Content-Type: application/json" -d "{\"model_path\":\"models/svsp.pt\"}"
```

Disable SVSP:

```powershell
curl -X POST http://localhost:8000/api/v1/model/svsp/disable
```

## Current Limitation

SVSP predicts motion direction from bbox history only.

It does not yet solve:

- open-hand gesture locking
- hand pose recognition
- explicit gesture-triggered target selection

Those require a separate future model.

## Best Practices

- train YOLO on real robot-camera images
- use correct object labels
- keep tracking stable
- test SVSP in demo mode first
- later replace synthetic SVSP training with real trajectory data
